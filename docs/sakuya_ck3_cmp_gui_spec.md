## 3. 主窗口

主窗口设计基准：

`1280 × 720`

外尺寸锁定为 720p 16:9。

主窗口是整个修改器的主要运行时窗口。

要求：

- 覆盖完整设计画布；

- 关闭按钮位置稳定；

运行时顶层窗口只保留：

1. 主窗口；
2. 入口按钮。

## 4. 首页结构

首页包含三个入口卡片：

1. 角色；
2. 头衔；
3. 家族。

三个入口点进去后进入三个不同风格的子页面。

三页互相不依赖，允许使用不同壁纸、装饰和局部视觉语言。

三入口仍应保持统一的：

- 尺寸体系；
- 边框逻辑；
- 暗金色标题；
- Hover 逻辑；
- 裁切逻辑；
- 光效；
- 点击反馈；
- 页面切换。

建议三个入口分别拆为独立组件文件，再由首页组合。

### 5.1 目标角色

优先级：

1. 已标记角色；
2. 玩家当前操控角色；
3. 无有效角色。

### 5.2 显示内容

角色入口至少显示：

- 背景图；
- 目标角色；
- 入口标题；
- 边框；
- 阴影；
- Hover 光效。

- 不显示角色模型；

角色应成为画面主体。

要求：

- 角色模型被卡片容器严格裁切；
- 身体和头部构图合理；
- 不允许角色穿出边框；
- 边框在角色和图片上层；
- 卡片边缘存在轻微阴影；
- 阴影不应过浓；
- Hover 放大时裁切仍然有效；
- 如 CK3 具有更适合的人物 portrait 裁切技术，应优先采用。

## 10. 模糊要求

模糊属于本轮重要视觉要求，应尽可能实现。

不得因为没有直接发现 `blur` 参数就立即放弃。

在决定无法实现前，必须调查：

- CK3 原生 GUI 中是否存在 blur；
- CK3 原生窗口是否使用背景模糊；
- CK3 是否存在可复用 shader、material、effect、mask 或 texture 组合；
- V3 CMP 的模糊实现；
- EU5 CMP 的模糊实现；
- CK3 是否可以使用预处理模糊图；
- CK3 是否可以在 Hover 状态切换清晰图和模糊图；
- 是否可以叠加低清晰度副本；
- 是否可以通过半透明柔焦蒙版模拟失焦；
- 是否可以通过多层缩放、透明度和遮罩模拟景深；
- 是否可以使背景模糊，同时保持角色或纹章清晰；
- 是否可以用局部光晕和清晰主体制造相近视觉结果。

### 10.1 可接受实现

以下均可视为模糊实现：

1. 实时模糊；
2. 原生 shader 模糊；
3. 预先制作模糊 DDS；
4. 清晰图与模糊图切换；
5. 模糊底图加清晰主体；
6. 低清晰度图片副本；
7. 柔焦纹理叠加；
8. 透明遮罩加中心焦点；
9. 能合理制造失焦和景深感的稳定组合。

### 10.2 模糊目标

Hover 时应产生：

- 背景失焦；
- 主体更加突出；
- 视线集中到角色、头衔或家族视觉中心；
- 卡片层次增强。

### 10.3 降级条件

只有经过实际代码和资源调查，确认没有稳定方案，才允许放弃模糊。

允许放弃的情况：

- CK3 不支持相关渲染；
- 需要覆盖原版 shader；
- 需要修改游戏安装目录；
- 会显著破坏兼容性；
- 会导致卡片裁切失效；
- 会造成严重性能或加载问题；
- 静态环境无法建立可信实现。

放弃时必须：

- 记录调查过的方案；
- 说明每种方案的问题；
- 使用中心放大、亮度变化、蒙版、柔光和边框高亮组成替代焦点效果；
- 在最终报告中明确说明。

不得用一句“CK3 可能不支持”直接跳过。

## 11. 中心放大

如果模糊无法完整实现，中心放大属于主要替代方案。

即使模糊已经实现，也可以保留适度中心放大。

要求：

- 放大中心位于视觉主体；
- 先尝试约 105%–110%；
- 需要更强效果时可以提高；
- 上限为 120%；
- 放大不能改变外部卡片尺寸；
- 放大不能突破裁切；
- 放大不能遮挡标题或关键按钮；
- 放大时边框仍然覆盖在图片上层。
