## 5. 文件命名规范

所有新建项目文件必须以 `sakuya_` 开头。

统一形式：

`sakuya_<category>_<section>_<name>_<suffix>.<ext>`

规则：

- 只使用小写 ASCII 字母；
- 可以使用数字；
- 使用下划线分隔；
- 禁止空格；
- 禁止中文文件名；
- 禁止连字符；
- 分页统一写作 `p1`、`p2`、`p3`；
- 后缀无实际用途时可以省略；
- CK3 引擎强制要求固定文件名时允许例外；
- 所有例外必须在最终报告中说明。

适用范围：

- `.gui`
- `.txt`
- `.asset`
- `.gfx`
- `.dds`
- `.png`
- `.yml`
- 当前项目中新建的其他资源文件

推荐分类：

- `main`
- `page`
- `character`
- `title`
- `dynasty`
- `common`
- `component`
- `animation`
- `background`
- `frame`
- `button`
- `icon`
- `localization`
- `test`

示例：

- `sakuya_main_window.gui`
- `sakuya_main_launcher_button.gui`
- `sakuya_page_character_p1_content.gui`
- `sakuya_page_title_p2_content.gui`
- `sakuya_page_dynasty_p3_content.gui`
- `sakuya_common_button_primary.gui`
- `sakuya_animation_page_hover.gui`
- `sakuya_background_character_p1.dds`
- `sakuya_localization_l_simp_chinese.yml`

目录名称无需全部使用 `sakuya_` 前缀，但目录必须清晰、稳定和语义化。

## 6. GUI 组件 `name` 规范

只有需要被引用、控制、定位或调试的组件设置 `name`。

需要设置 `name` 的情况：

- 脚本需要访问；
- 需要显示或隐藏；
- 需要切换状态；
- 需要绑定动画；
- 需要作为动画目标；
- 需要作为父容器；
- 需要调试识别；
- 需要被其他组件引用；
- 需要保存页面状态；
- 需要鼠标检测。

纯装饰元素且不参与引用时，可以省略 `name`。

统一形式：

`sakuya_guiname_<category>_<name>[_<suffix>]`

示例：

- `sakuya_guiname_main_window`
- `sakuya_guiname_main_page_selector`
- `sakuya_guiname_character_card`
- `sakuya_guiname_character_portrait`
- `sakuya_guiname_title_card_hover`
- `sakuya_guiname_dynasty_page_return_button`

禁止使用：

- `box1`
- `button2`
- `widget3`
- `test1`
- `new_button`
- `temp_widget`
- 其他无语义名称

除非存在明确技术原因，已投入使用的组件名称应保持稳定。

## 7. 本地化规范

当前阶段仅提供简体中文。

所有新增 `.yml` 文件：

- 文件名必须以 `sakuya_` 开头；
- 使用 CK3 支持的正确编码；
- 使用正确的简体中文语言头；
- 尽量复用文本；
- 尽量复用 CK3 原生本地化键；
- 尽量使用 `$KEY$` 串联；
- 不重复创建含义相同的键；
- 不为纯装饰元素创建无意义文本。

跨页面高频公共键使用：

`SKY_XXX`

建议键：

- `SKY_L`：左键
- `SKY_R`：右键
- `SKY_CL`：Ctrl + 左键
- `SKY_CR`：Ctrl + 右键
- `SKY_SHIFT_L`：Shift + 左键
- `SKY_SHIFT_R`：Shift + 右键
- `SKY_sakuya`：咲夜
- `SKY_cmp`：喜儿修改器
- `SKY_close`：关闭
- `SKY_return`：返回
- `SKY_character`：角色
- `SKY_title`：头衔
- `SKY_dynasty`：家族
- `SKY_no_character`：无有效角色
- `SKY_no_title`：无有效头衔
- `SKY_lowborn`：出身卑微
- `SKY_test`：测试
- `SKY_placeholder`：占位

其他本地化键应采用清晰、统一、可追踪的项目命名。

术语优先复用 CK3 原生键。只有原生键含义不准确、语法不适用或无法稳定引用时，才创建自有键。

## 9. 分辨率与画布

设计基准：

`1280 × 720`

主窗口外尺寸锁定为 720p 16:9，与当前 CK3 整个窗口大小一致。

要求：

- 主要元素在 1280×720 下完整显示；
- 不超出安全区域；
- 不依赖高分辨率掩盖问题；
- 图片、人物、纹章、动画不得穿出边框；
- 文字不得越界；
- 按钮不得重叠；
- 更高分辨率下保持合理比例；
- UI Scale 变化时尽量可用；
- 优先使用 CK3 支持的相对定位、anchors、尺寸继承和缩放。

## 11. 页面状态

至少包含以下状态：

- 首页；
- 角色页；
- 头衔页；
- 家族页。

要求：

- 用户从首页进入对应子页面；
- 关闭修改器时只关闭窗口；
- 关闭时不重置当前页；
- 再次打开时尽可能恢复上次停留页面；
- 每个子页面都必须有关闭按钮；
- 关闭按钮右侧必须有返回按钮；
- 返回按钮回到三入口首页；
- 返回按钮不关闭修改器；
- 关闭按钮不改变页面状态。

若 CK3 GUI 架构无法可靠保留上次页面，应先寻找状态变量、scripted GUI、可见性条件或其他稳定办法。最终仍无法实现时才允许降级，并在报告中明确说明。
